== CORS and CSP

=== Refused to frame "<your-site-url>"... error message

[width="100%" cols="1,6"]
|====
|**Error**|  When I try to embed in my app, I get the error `Refused to frame "<your-site-url>" because an ancestor violates the following Content Security Policy directive`.
|**Cause**|  This error occurs when a host app domain is not added to CSP and CORS allowlists.
|**Solution** a|  Perform the following checks: +

* Check if xref:security-settings.adoc[you have added set your host app domain as a CSP visual embed host]. +
* Check if you have xref:security-settings.adoc[enabled CORS] for your host application.
|====

=== The wildcard (*) for domain URL in the CORS allowlist returns an error

[width="100%" cols="1,6"]
|====
|**Error**| The CORS allowlist does not allow wildcard(*) expression
|**Cause**|  This error occurs because ThoughtSpot does not support wildcard expression for domain URLs in the CORS allowlist.
|**Solution** a| Specify the domain URL string in the supported format. For a complete list of allowed domain URL strings for CORS and CSP allowlists, see xref:security-settings.adoc#csp-cors-hosts[Security settings].
|====

== Browser support

=== Cannot open my embedded app in Safari

[width="100%" cols="1,6"]
|====
|**Error**| Unable to access embedded app on Safari
|**Cause**| This error may occur if third-party cookies are blocked.
|**Fix** a| By default, Safari does not allow third-party cookies. If the third-party cookies are blocked, the SDK triggers the `NoCookieAccess` event. Check the console log. +
To resolve this issue: +

* Allow cookies on Safari to view the embedded app. +
* To ensure that embedding works on browsers that don't allow third-party cookies, you may want to set the domain name of the ThoughtSpot instance to the same domain as your host application. To customize the ThoughtSpot domain name, contact ThoughtSpot Support.
|====

== Custom actions

=== URL custom actions don't work on my app

*The URL custom actions don't work in my app*

* Check the target domain name configured in the URL action. Make sure it's reachable.
* Check if the target URL configured in the URL action is added to CSP connect src domain allowlist.

=== Unable to embed a Liveboard using REST APIs

*I am unable to embed a Liveboard using REST API in my app*

* Check if you have edit access to the Liveboard you are embedding
* Check if you have enabled CORS for your host application domain

=== The Search Data REST API call returns an error 

*When I try to use the Search Data REST API, I get the error "Cannot create answer from empty query"*.

* Check the request URL and verify the query string.
* Try removing the `-d [parameters]` option and append the query string to the end of the URL as shown here:  

+
[source,Javascript]
----
curl -X POST /
--header 'Content-Type: application/json' /
--header 'Accept: application/json' /
--header 'X-Requested-By: ThoughtSpot' 'https://[address]/callosum/v1/tspublic/v1/searchdata?query_string=%5Brevenue%5D%20by%20%5Bshipmode%5D&data_source_guid=7466f3c5-95a5-44b9-a17d-9cc5fad495ee&batchsize=-1&pagenumber=-1&offset=-1&formattype=COMPACT' -b cookies.txt
----

=== Liveboard filters do not work

*When I open Liveboard filters, they are grayed out and I can’t change them*

This issue may occur if you do not have access to the Liveboard or its underlying data. 

* Check if you have view or edit access to the data source.
* If you do not have edit access to the Liveboard, click the lock icon and request access.

=== ThoughtSpot navigation bar shows in embedded mode

*When I embed full ThoughtSpot application, the top navigation bar shows*

To hide the top navigation bar, set the `showPrimaryNavbar` attribute in the Visual Embed SDK to `false`.   


=== Token based authentication returns a 401 error

* Are you including the key in the Security Settings in the Developer portal?
* Has that key changed from the last time you included it


=== SSO authentication does not work

* Are all the SAML settings in the SAML section of the Admin tab correct?
* Did you set the SAML redirect domain in the Security Settings in the Developer portal?
* Are all the settings in your IDP provider correct?


=== I want to show only a few actions in the Liveboard menu



.I set a Runtime filter on a Liveboard, but I don’t see it on the Liveboard
[%collapsible]
====

* Runtime filters do not show up in the Filter bar, but are applied

*If there is already a Filter on a Field, and then I set a Runtime Filter, what happens?*

* Runtime filters are applied separately from the visible Pinboard filters, so both conditions must be true when combined together as an AND for data to appear
* Ex. If “Store Region” is set to “west, southwest” in the Pinboard filter, and you set Runtime Filter of “Store Region” : [‘east’], you will get *No data found* because the total filter condition, which cannot ever be true, is: Store Region IN (‘west’, ‘southwest’) AND Store Region = ‘east’
====
