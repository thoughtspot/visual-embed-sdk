:page-title: Integrate ThoughtSpot with Slack app
:page-pageid: slack-integration
:page-description: You can connect users to their Slack workspace via app actions and allow them to push insights to a Slack channel 

With most buisnesses using Slack for inter-team communication, ThoughtSpot users may want to deliver insights directly to Slack channels in their workspace. Using ThoughtSpot app actions, business users and analysts can connect their Slack workspace to ThoughtSpot and push insights in a few clicks.  

== Prerequisites

* A ThoughtSpot application instance with *New Answer Experience* is enabled
* A ThoughtSpot user account with developer or admin privileges
* A business user account that has access to a valid Slack workspace

== Workflow summary

The app integration workflow involves the following steps:

. A developer or admin user creates an app action for Slack in the Developer portal. This step sets Slack as a destination app.
. A user clicks the app action and invokes the workflow to create a pipeline between ThoughtSpot and the destination app. This step initiates the OAuth workflow for a secure exchange of data between the source and destination app. 
. If the user is already logged in to the Slack workspace, ThoughtSpot prompts the user to authorize the app and allow access to the Slack workspace. 
. The user clicks **Allow**. ThoughtSpot obtains a list of channels to which the user belongs from the Slack workspace.
. The user selects a channel and clicks **Send now**.
. ThoughtSpot pushes the insights from the answer to the specified channel in the specified data format.

[#app-action-slack]
=== Create an app action with Slack as the destination app

Before you begin, make sure you are logged in as a ThoughtSpot user with developer or admin privileges.

To create an app action for Slack, follow these steps:

. Go to *Develop* > *Customizations* > *Custom actions*.
. Click *Create action*.
. Add a label for the custom action. For example, __Send to Slack__.
. Select the *App* option and then select **Slack**.
+
Note that the app action ID is generated automatically. 

. To add this action to all visualizations and saved answer pages, select *On by default on all visualizations*. 
+
If you do not select this checkbox, the app action will be set as a *Local* action.  

+
. To define which user groups can access the app action, click *Show advanced availability settings*, and select the user groups. By default, the app action will be available to all ThoughtSpot users.
 
. Click *Create action*.
+
The custom action is added to the *Actions* dashboard in the Developer portal.

. To view the custom action you just created, navigate to a visualization or saved answer page.

[#initiateActionSlack]
=== Initiate the app action

Before you begin, make sure you are logged in as a business user or analyst with your ThoughtSpot user credentials. 

To initiate a data push request to your Slack workspace, follow these steps:

. Go to the Liveboard visualization or the saved answer that has the insights you want to push to your Slack workspace.
. If you have created an answer from a new query, make sure you save the answer.
+
If the app action is set as a global action, the action will appear in the **More** menu image:./images/icon-more-10px.png[the more options menu]. If the action is set as a local app action, add it to your answer page.

. Click the action.
+
ThoughtSpot requests access to your Slack workspace and opens the app authorization page. This step is required only if you are using the Slack integration for the first time, or when ThoughtSpot refreshes its OAuth access token.

+
[NOTE]
====
The app authorization page opens as a pop-up, so make sure pop-ups are not blocked by your web browser.
====

. If your email address is connected to multiple Slack workspaces, select a workspace from the drop-down located at the top of the app authorization page and then click **Allow**.
. Wait for a few seconds for ThoughtSpot to connect to your workspace and obtain a list of channels. 

+
If the connection is successful, the app action dialog in the ThoughtSpot UI displays a list of Slack channels to which you can deliver data.

. Select a channel from *Slack channel* drop-down. 
+
You can also search for a channel and select it.

. If required, modify the title of the data and add a message.
. Specify if the data must be sent as a PNG file or in the CSV format, or both. 
+
You cannot send data from certain visualizations as an image.

[#viewInSlack]
=== View insights in Slack 

To view the insights sent to a Slack channel:

. Log in your Slack workspace.
. Go to the channel to which you have sent the data. 

You can also view the ThoughtSpot app from the *Apps* page in your Slack workspace.
